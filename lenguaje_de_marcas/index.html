    • Origenes Lenguaje de Marcas

    Los lenguajes de marcas son una forma de codificar un documento que, junto con el texto, incorpora etiquetas o marcas que contienen información adicional acerca de la estructura del texto o su presentación. El lenguaje de marcas más extendido es el HTML (HyperText Markup Language, lenguaje de marcado de hipertexto), fundamento del World Wide Web. Los lenguajes de marcas suelen confundirse con lenguajes de programación, pero no son lo mismo, ya que el lenguaje de marcado no tiene funciones aritméticas o variables, como poseen los lenguajes de programación. El marcado se usaba y se usa en la industria editorial y de la comunicación, así como entre autores, editores e impresores. Históricamente, el marcado se usaba para dirigir el proceso de presentación (impresión) mediante una linotipia o impresora programable.

    Autores HTML



    • William W. Tunnicliffe fue el primero en exponer en concepto de lenguaje de marcas como forma de representación de la información.

    • El investigador de IBM Charles F. Goldfarb fue el que expuso por primera vez en 1967 las ideas de William W. Tunnicliffe dando crédito a su investigación.

    • En 1969 Charles F. Goldfarb creó el estándar GML (Generalized Markup Language).

    • En la década de 1970, Tunnicliffe lideró el desarrollo de un estándar llamado GenCode para la industria editorial y más tarde fue el primer presidente de la Organización Internacional de Normalización.

    • En 1986 Charles F. Goldfarb creó la norma ISO 8879 bajo el nombre de SGML (Standard Generalized Markup Language). Es considerado hoy en día como el padre de los lenguajes de marcas ya que creó SGML que es el estándar en el que se basan los lenguajes XML y HTML.

    Diferentes estándares y organismos oficiales



    • ISO: La Organización Internacional de Normalización (ISO) es una organización no gubernamental que desarrolla y publica normas internacionales. En el ámbito de los lenguajes de marcas, la ISO ha desarrollado el estándar SGML (Standard Generalized Markup Language) y sus derivados, como HTML y XML.


    • W3C: El Consorcio World Wide Web (W3C) es una comunidad internacional que desarrolla estándares web abiertos. El W3C ha desarrollado varios estándares de lenguajes de marcas, como HTML, XHTML y XML.


    • IETF La Fuerza de Tarea de Ingeniería de Internet (IETF) es una organización que desarrolla y promueve estándares de Internet. La IETF ha desarrollado varios estándares de lenguajes de marcas, como HTML y XML.


    • ECMA: La Asociación Europea de Fabricantes de Computadoras (ECMA) es una organización que desarrolla estándares para la fabricación de productos electrónicos. La ECMA ha desarrollado varios estándares de lenguajes de marcas, como ECMAScript (también conocido como JavaScript).

    Orígenes y las diferencias entre CSS y SCSS.

    CSS (Cascading Style Sheets) es un lenguaje de hojas de estilo utilizado para describir la presentación de un documento HTML o XML. Fue desarrollado por el W3C y se ha convertido en el estándar de facto para la presentación de documentos web.

    SCSS (Sassy CSS) es una extensión de CSS que agrega características adicionales al lenguaje. SCSS es un preprocesador de CSS que permite a los desarrolladores escribir CSS de manera más eficiente y organizada. SCSS es compatible con todas las versiones de CSS y se compila en CSS válido.

    La principal diferencia entre CSS y SCSS es que CSS es un lenguaje de hojas de estilo plano, mientras que SCSS es un lenguaje de hojas de estilo anidado. Esto significa que en SCSS, los estilos se pueden anidar dentro de otros estilos, lo que hace que el código sea más fácil de leer y mantener. Además, SCSS admite variables, funciones y mixins, lo que permite a los desarrolladores escribir CSS de manera más modular y reutilizable.

    En resumen, CSS es un lenguaje de hojas de estilo plano que se utiliza para describir la presentación de documentos web, mientras que CSS es una extensión de CSS que agrega características adicionales al lenguaje y permite a los desarrolladores escribir CSS de manera más eficiente y organizada. La principal diferencia entre CSS y SCSS es que SCSS es un lenguaje de hojas de estilo anidado que admite variables, funciones y mixins.

    Principales etiquetas HTML5

    HTML5 es la última versión del lenguaje de marcado HTML. Las etiquetas HTML son elementos que se utilizan para definir el contenido y la estructura de una página web. Las etiquetas semánticas son un conjunto de etiquetas HTML que se utilizan para describir el contenido de una página web de una manera más significativa y estructurada. Algunas de las etiquetas semánticas más comunes en HTML5 son:

    header: Define una sección de encabezado para la página o una sección.

    nav: Define una sección de navegación para la página.

    article: Define una sección de contenido independiente y autónomo.

    section: Define una sección de contenido temático.

    aside: Define una sección de contenido relacionado pero no esencial.

    footer Define una sección de pie de página para la página o una sección.

    Las etiquetas semánticas ayudan a los motores de búsqueda y otros agentes de usuario a comprender mejor el contenido de una página web. También pueden mejorar la accesibilidad y la usabilidad de una página web para los usuarios.

    Principales elementos CSS

    En CSS, los elementos son los bloques de construcción básicos que se utilizan para diseñar una página web. Algunos de los elementos más comunes son: Color: se utiliza para establecer el color de fondo o de primer plano de un elemento. Tipografía: se utiliza para establecer el tipo de letra, el tamaño y el estilo de un elemento. Espaciado: se utiliza para establecer el espacio entre los elementos. Bordes: se utiliza para establecer el ancho, el estilo y el color de los bordes de un elemento. Posicionamiento: se utiliza para establecer la posición de un elemento en la página. Fondo: se utiliza para establecer una imagen o un color de fondo para un elemento.
